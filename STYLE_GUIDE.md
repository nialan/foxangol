# FoxAngol Style Guide

"Consistency makes content easier to read, maintain, and improve."

## Purpose

This document defines the editorial and technical conventions used in the **FoxAngol** project.

The goal is to ensure consistency across all pages and blog posts.

---

## Editorial Style

### Tone

- Friendly
- Professional
- Encouraging
- Accessible
- Personal, but not overly informal

### Headings

- One H1 per page
- Use H2 for major sections
- Use H3 only when necessary

### Paragraphs

- Prefer short paragraphs
- Maximum 4–5 lines
- One idea per paragraph

### Images

- Always include alt text
- Width: 400 px
- Center aligned
- Store images in:

blog/posts/images/

### File names

Use lowercase letters and hyphens.

Good:

miben-segit-ha-idegen-nyelven-olvasol.png

Avoid:

Miben Segít!.png

---

## Technical Style

### Branch naming

- feature/add-blog-post
- feature/update-about-page

### Commit messages

Use English.

Examples:

- Add About page content
- Update lesson types
- Fix image path

### Markdown

- Use Markdown whenever possible
- Use HTML only for image formatting
- Keep formatting simple

### Navigation

Every new page must be added to mkdocs.yml.
